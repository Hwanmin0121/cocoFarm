직렬화 1,2,3 : 임시 영수증의 처리 대상이 경매와 일반 판매 두가지로 나뉘어 있고, 대상의 수 와 목록의 형태가 어떻게 들어와도 처리하기 위해 db에 처리할 데이터를 전달할 직렬화 형식을 정함. 그리고 그것을 구현할 인터페이스를 구현하고 abstrack default 메소드를 이용해서 다른 팀원의 코드에 영향을 취소화.

직렬화 4: 본격적인 직렬화 처리를 위해 method 들을 overload 처리해 놓고 한 개의 클래스에 static 메소드로 구현하여, 객체의 생성 없이 필요한 데이터들을 바로 직렬화 하게 함.

직렬화 5: 여기도 다양한 데이터 입력 형태를 처리하기 위해 overload 해 놓은 메소드들 중 일부. DAO 에 직렬화 한 데이터를 RecptCallParamHolder 에 담아 메소드를 호출하고 결과를 내보내는 서비스 메소드.

직렬화 6: 직렬화한 데이터를 받기 위해 varray 를 준비하고 있는 procedure.

직렬화 7: 직렬화한 데이터를 해석.
